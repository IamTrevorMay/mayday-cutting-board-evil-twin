Cutting Board Evil Twin

Build steps 2-4 validation harness
Test 1 — Silence Remover bridge (Step 2 gate)
Tells Evil Twin to call into the Silence Remover plugin and asks it to analyze whatever's in the active sequence. Pass = a list of silent regions comes back.
Requires: Silence Remover plugin installed + active sequence with audio.
Test 2 — Transcribe a file (Step 3)
Asks the SDK to transcribe an audio/video file with whisper. Pass = segments with timestamps come back. Use a short clip (under a minute) for a fast test.
Requires: whisper-cli on PATH and ggml-base.en.bin model installed.
Test 5 — Real audio: transcribe + detect takes
Pipes a real audio/video file through whisper, then runs the take detector on the transcript. Use a recording where you (or someone) restarts a sentence at least once — the detector won't have anything to find on a clean read. Long files take a while; under 2 minutes is comfortable.
Result shows each cut range with the text being cut, so you can sanity-check.
Test 4 — Take detector (Step 5, the actual idea)
Runs the heuristic against a built-in sample transcript with deliberate restarts. The sample includes 13 segments where the speaker re-starts a few times. Pass = the algorithm groups the restarts together and identifies which take to keep vs. cut.
No Premiere needed for this one — pure algorithm test.
Test 3 — Duplicate + range ripple delete (Step 4 gate, CRITICAL)
Duplicates the active sequence then ripple-deletes a chosen range across all tracks on the duplicate. Open the duplicate after — clips after the range should have shifted back by exactly (end - start) seconds with audio still in sync with video.
Click the Premiere Timeline panel first so it has focus, then run.
